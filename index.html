📝 De un Vistazo: Comando Central

Todo lo que necesitas para preparar y ejecutar esta misión en minutos.

🎯 Objetivo de Aprendizaje

Los estudiantes podrán: comprender la integración como un contrato y la trazabilidad de datos. Usar WebWave y JDoodle para implementar peticiones fetch (POST).

🤖 Prompt de Referencia para XOLO (Solo para Maestro)

⚠️ NOTA: XOLO ya conoce el tema de esta misión. Este prompt es SOLO de referencia en caso de emergencia técnica: 
Explícame cómo el frontend “habla” con el backend mediante fetch y qué elementos componen una petición: método (POST), headers, body y endpoint. Analicemos qué datos debo enviar desde un formulario, cómo serializarlos (JSON) y cómo verificar que la petición realmente salió (sin asumir). Guíame a escribir el fragmento de JS que conecta mi UI con un endpoint real del backend.

⚡ Puntos Esenciales de la Misión

🎓

Concepto Clave

Integración como contrato: Es un acuerdo entre el frente de la página y el cerebro del servidor sobre qué datos se envían y cómo se reciben, asegurando que ambos hablen el mismo idioma.

🔧

Habilidad Técnica

Usar código JavaScript para enviar información desde un formulario web hacia un servidor externo.

🧪

Experimentación

Probar con diferentes estructuras de datos en el cuerpo del mensaje (body) y observar cómo responde el servidor.

💬

Interacción con XOLO

XOLO ya conoce a cada estudiante y el tema de hoy. Solo abren el chat y comienzan a aprender. Tú guías el timing y motivación.

🚨 Resolución Rápida de Problemas

  • Si los alumnos no pueden ingresar a Gnius Space: Activa XOLO SOS en xolo.gnius.club para continuar la misión sin interrumpir la clase.
  • Si la petición fetch falla: Revisar que el endpoint (URL) sea correcto y que el servidor esté activo. Verificar en la consola del navegador (F12) si hay errores de CORS.
  • Si el JSON no se envía bien: Asegurarse de usar JSON.stringify() en el body y que el header 'Content-Type' sea 'application/json'.
  • Si hay errores en el código: Recordar a los estudiantes que XOLO puede ayudar a depurar. Pídeles que copien el error y se lo muestren a XOLO.

📊 Diapositiva 3: Brain Spark

1. 💬 Pide a XOLO que te ayude a crear:

"Ayúdame a integrar mi frontend con mi backend con Vibe Coding: define conmigo el botón/acción, qué datos recolecto (formulario mínimo) y a qué endpoint haré POST. Guíame a construir el fetch con body JSON y un “log de comunicación” básico. Ejecuta: pega el fragmento JS en mi frontend, realiza una prueba manual (clic) y registra qué datos enviaste y a qué ruta."

2. 🔬 Experimenta y descubre con tu programa:

  • ¿La petición llega al servidor y recibes una respuesta exitosa (status 200)?
  • ¿Qué sucede si intentas enviar el formulario con campos vacíos o datos incorrectos?
  • ¿Funciona igual la petición si cambias el método de POST a GET?
  • ¿Cómo asegura el 'contrato de integración' que el frontend y el backend no se confundan al enviarse información?
  • Si los datos son trazables, ¿por qué es nuestra responsabilidad enviar solo lo mínimo necesario?

¡Es hora de crear! Ya entendieron la teoría, ahora vamos a la práctica. Usaremos XOLO para idear un proyecto donde un formulario en su web envíe datos reales a un endpoint. Imaginen, por ejemplo, un sistema de registro para un evento o un buzón de sugerencias inteligente. Sigan el flujo: ideen con XOLO, definan el 'VIBE' de su comunicación, obtengan su código GCS, llévenlo a WebWave o JDoodle y experimenten. No olviden hacerse las preguntas de experimentación para ver si su 'puente' es sólido y si los datos fluyen como esperaban. Si tienen errores o dudas, pregúntenle a XOLO, él los ayudará a depurar el código de inmediato. ¡Manos a la obra, genios del código!
